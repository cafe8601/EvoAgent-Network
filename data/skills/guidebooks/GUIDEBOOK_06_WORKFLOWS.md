## 🟢 기본 워크플로우 (Basic)

### 예시 1: 처음부터 한국어 LLM 만들기

```
1. [02-tokenization] 한국어 토크나이저 학습 (HuggingFace/SentencePiece)
2. [05-data-processing] 한국어 웹 데이터 정제 및 중복 제거
3. [01-model-architecture] 모델 아키텍처 설정 (Llama-3 기반)
4. [08-distributed-training] DeepSpeed/FSDP로 대규모 분산 사전 학습
5. [03-fine-tuning] 한국어 지시문 데이터(Ko-Alpaca 등)로 SFT 수행
6. [06-post-training] DPO로 한국어 선호도 및 안전성 정렬
7. [11-evaluation] LogicKor, Ko-MMLU 등 한국어 벤치마크 평가
8. [12-inference-serving] vLLM으로 OpenAI 호환 API 배포
```

### 예시 2: RAG 기반 사내 Q&A 시스템

```
1. [05-data-processing] 사내 문서(PDF, Wiki) 수집 및 청킹
2. [15-rag] ChromaDB로 문서 임베딩 및 인덱싱
3. [14-agents] LangChain으로 검색 및 답변 생성 체인 구현
4. [16-prompt-engineering] DSPy로 답변 생성 프롬프트 자동 최적화
5. [17-observability] LangSmith로 검색 품질 및 답변 정확도 모니터링
6. [07-safety-alignment] 기밀 정보 유출 방지 가드레일 추가
7. [09-infrastructure] Modal 서버리스 함수로 전체 파이프라인 배포
```

## 🔵 멀티에이전트 심화 (Multi-Agent)

### 예시 12: 실시간 고객 지원 멀티에이전트 🎧

```
고객 문의 → 분류 → 전문 에이전트 → 응답 생성 → 품질 검증

1. [21-multiagent] 고객 지원 에이전트 풀 구성
   - TierRouter: 문의 복잡도 및 유형 분류
   - ProductExpert: 제품 기능 상세 답변
   - BillingAgent: 결제, 환불, 요금제 처리
   - TechnicalSupport: 버그 리포트 및 기술 지원
   - EscalationAgent: VIP 고객 또는 해결 불가 시 인간 연결

2. [15-rag] RAG 지식 베이스 연동
   - FAQ, 매뉴얼, 과거 상담 이력 검색

3. [21-multiagent] 학습 및 메모리 시스템
   - MemoryManager: 이전 상담 내역 기억 (Context 유지)
   - LearningManager: 해결된 상담 케이스 학습하여 라우팅 개선

4. [07-safety-alignment] 고객 보호 가드레일
   - PII 필터링 (전화번호, 카드번호 마스킹)
   - 감정 분석을 통한 불만 고객 조기 감지
```

### 예시 13: 코드 리뷰 자동화 파이프라인 🔍

```
PR 생성 → 코드 분석 → 다중 관점 리뷰 → 통합 피드백 → 승인/수정

1. [21-multiagent] 리뷰어 에이전트 팀 구성
   - SecurityGuard: 보안 취약점(SQL Injection 등) 점검
   - StyleCop: 코딩 컨벤션 및 스타일 가이드 준수 확인
   - PerformanceGuru: 알고리즘 효율성 및 병목 구간 진단
   - TestArchitect: 테스트 커버리지 및 테스트 케이스 누락 확인
   - ArchitectureReviewer: 설계 원칙 위반 여부 검토

2. [14-agents] GitHub API 연동
   - PR 내용 파싱 및 변경된 파일 추출
   - 라인별 코멘트 자동 작성

3. [21-multiagent] 리뷰 통합 (Aggregator)
   - 중복된 지적 사항 통합
   - 중요도(Blocker, Critical, Minor)에 따라 우선순위 정렬
   - 최종 리뷰 코멘트 요약 생성
```

## 🟣 블록체인 통합 (Blockchain)

### 예시 22: AI 에이전트 신원 인증 블록체인 시스템 🔐

```
에이전트 생성 → DID 등록 → 자격 검증 → 작업 수행

1. [22-blockchain] 에이전트 신원 레지스트리 (Smart Contract)
   - AgentIdentityRegistry 배포
   - 에이전트 고유 DID 생성 및 메타데이터(버전, 기능) 해시 등록
   - 소유자 지갑 주소 매핑

2. [21-multiagent] 에이전트 부트스트랩
   - AgentPoolManager가 새 에이전트 생성 시 자동으로 DID 등록 요청
   - 초기 보증금(Stake) 예치 (신뢰도 확보)

3. [22-blockchain] 자격 증명 (Verifiable Credentials)
   - '금융 거래 가능', '개인정보 접근 가능' 등 자격(VC) 발급
   - 작업 요청 시 ZK Proof로 자격 증명 제출 (프라이버시 보호)

4. [17-observability] 온체인 활동 모니터링
   - 에이전트의 모든 주요 활동을 트랜잭션으로 기록
   - 이상 징후 감지 시 자동으로 일시 정지(Circuit Breaker)
```

### 예시 26: AI 에이전트 마켓플레이스 시스템 🛍️

```
서비스 등록 → 검색 및 매칭 → 서비스 이용 → 결제 및 평판

1. [22-blockchain] 마켓플레이스 컨트랙트
   - 에이전트 서비스(번역, 분석, 코딩 등) 목록화
   - 이용료(Token) 설정 및 에스크로 기능

2. [21-multiagent] 서비스 제공자 에이전트
   - 대기 상태에서 요청 리스닝
   - 요청 수신 시 작업 수행 및 결과물 해시 온체인 기록

3. [22-blockchain] 결제 및 정산
   - 작업 완료 검증 후 에스크로된 토큰 자동 지급
   - 플랫폼 수수료 공제

4. [22-blockchain] 평판 시스템
   - 사용자가 결과물에 대해 평점 부여 (온체인 기록)
   - 평판 점수에 따라 검색 노출 순위 및 이용료 조정
```

## 🟠 도메인 특화 (Domain)

### 예시 32: 의료 AI 진단 보조 시스템 🏥

```
의료 영상/차트 → 데이터 전처리 → AI 분석 → 의사 검토

1. [18-multimodal] 의료 데이터 처리
   - X-ray, MRI 등 DICOM 이미지 파싱
   - 의무기록(텍스트) 비식별화 및 구조화

2. [03-fine-tuning] 의료 특화 모델 학습
   - PubMed, 의료 교과서 등으로 LLM 미세조정 (Medical-LLM)
   - 의료 영상 데이터로 Vision Encoder 학습
```

   - SplitExecutioner: 충격을 줄이고 평단가를 관리하기 위한 분할 매수/매도 스케줄링
